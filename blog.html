1. What are the differences between Local Storage and Session Storage ??

The differences between Local Storage and Session Storage are mentioned bellow

Local Storage

  1. Data in local storage does not expires automatically. It has to be done manually.
  2. The Storage Capacity in Local Storage is about 10 MB.
  3. There is no transfer of data to the server.
  4. The client can only read local storage data.
  5. Its data is never transferred to the server

Session Storage

  1. Data in session storage expires after the end of session. like (closing the browser)
  2. The Storage Capacity in Session Storage is 5 MB.
  3. There is no transfer of data to the server.
  4. The client can only read local storage data.
  5. Its data is never transferred to the server

2. What are the differences between Global Scope and Block scope in JavaScript ??

The differences between Global Scope and Block scope in JavaScript are :

Global Scope

  1. The variables defined outside of any function or curly brackets { } are known as global variables and The scope the have is called global scope
  2. Variables that are declared in global scope can be accessed from anywhere in JavaScript Code
  3. If we declare variable with var from anywhere of the code it goes to global Scope.
  4. The area outside of any function and curly braces { } is global scope in JavaScript.

Block Scope

  1. Generally, whenever we use curly brackets { } in code it creates a block and variables inside the block are in block scope.
  2. Variable that are declared inside a block scope can only be accessed inside that block
  3. ES6 provides the let and const keywords that allow us to declare variables in block scope.
  4. The area within the if, else, switch ,conditions or for, do while, and while loops is block scope.

3. How Does Event Loop Works In JavaScript ??

Event Loop

The Event loop is the secret behind JavaScript's asynchronous programming.

Event Loop does an important job . It monitors the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop. Each event is just a function callback.

4. How many Ways To Get Undefined In JavaScript ??

We might get Undefined in JavaScript for the following 8 reasons :

  1. Variable that is not initialized will give undefined.
  2. Function with no return will give undefined when it is called.
  3. If Parameter is not passed to a function when if is necessary will give undefined.
  4. If return has nothing on the right side will return undefined.
  5. Trying to access a Property that does not exist inside a object will give undefined.
  6. Accessing array elements outside of the index range will give undefined
  7. After Deleting an element inside of array if we try to access it , we get undefined.
  8. If we Set a value directly to undefined . We will get undefined.
